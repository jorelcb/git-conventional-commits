# Contributing to git-conventional-commits

Thank you for your interest in contributing! This project follows a standard open-source workflow.

## Development Workflow

1.  **Fork** the repository on GitHub.
2.  **Clone** your fork locally.
3.  Create a **Feature Branch** (`git checkout -b feature/my-feature`).
4.  Make your changes following the **coding standards** (see below).
5.  **Test** your changes (`npm run build`).
6.  **Push** to your fork.
7.  Submit a **Pull Request** to the `main` branch.

## Coding Standards

- **Language**: TypeScript (Strict mode).
- **Commits**: MUST follow [Conventional Commits](https://www.conventionalcommits.org/).
    - Example: `feat: add new validation rule`
    - Example: `fix: resolve issue with git path`
- **No AI Credits**: As per project rules, avoid adding "Generated by AI" to commits or files.

## Project Structure

- `commands/`: Gemini CLI slash command definitions (`.toml`).
- `GEMINI.md`: Context for AI agents using this tool.
- `gemini-extension.json`: Extension Manifest.
